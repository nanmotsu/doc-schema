ノートを作成、編集する際に以下の選択肢がある
- vsCode上で行う
- obsidian上で行う。
- 独自アプリ上（graph.html, graph.mjs）で行う

それぞれのメリットデメリットをいかにまとめる

|          | メリット                                 | デメリット                                                                          |
| -------- | ------------------------------------ | ------------------------------------------------------------------------------ |
| vsCode   | スニペットが使用できる<br>Node＋Taskを用いてカスタムしやすい | 編集・閲覧を切り替えないといけない<br>foamのグラフビューが使いにくい（フィルター）<br>（パス指定はsetting.jsonにかかないといけない） |
| obsidian | ビュー＝編集画面<br>グラフビューの柔軟性               | スニペットが使えない<br>テンプレについてjs直書きのためごちゃつく                                            |
| 独自アプリ    | 自由度が高い                               | 開発や保守のコストかかる                                                                   |


## ガイドライン
それぞれ使い分ける

**vsCode**
	- 集計やチェックのスクリプトを走らす時
	- 新規作成
	- Statusの変更を伴う編集
	- drawio Excalidrawの作成 png保存
**obsidian**
	- スニペットや画像やStatusを扱わない改修
独自アプリ
	- 全体を確認（フィルターや検索可能）
	- Statusを変える

画像を載せる時は pngやjpegなどの形として、
作成元データ（drawio Excalidraw）を残しておく。
（元データは`.drawio`みたいなコンマで始まるディレクトリに格納することでobsidian上では扱えないようにする）
