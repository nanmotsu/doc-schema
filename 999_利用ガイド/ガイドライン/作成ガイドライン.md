## 全体の流れ
基本的にSource起点


各名称とプレフェックス

また、終了後
「runbook」や「trouble_shooting」を作成、更新し、specificationからリンクを貼っておく
### 0. 共通
```
source → issue → decision → requirement → specification 
                                ↑
							   spec → task（test）
```

### 1. 新規開発
```
source（要望・企画）
↓
issue（曖昧さ整理）
↓
decision（方向決定）
↓
requirement（外部に出す）
↓
specification（外部仕様）
↓
spec（ロジック）
↓
task（実装 ＋ テスト）
```

### 2. 顧客要件が最初からある
```
source（顧客要件）
↓
requirement（仮 ドラフト）
↓
issue（矛盾・不足）
↓
decision（解釈）
↓
requirement（再定義）
↓
specification（外部仕様）
↓
spec（ロジック）
↓
task（実装 ＋ テスト）
```

### 3. 既存機能の改修
```
source（既存仕様・コード）
↓
issue（問題点）
↓
decision（改善方針）
↓
requirement（改善要件 「差分」で書く）
↓
specification Or spec（変更仕様）
↓
task（実装 ＋ テスト）
```

### 4. バグ修正
requirementが省略されることも多い
```
source（ログ・障害）
↓
issue（原因）
↓
decision（修正方法）
↓
specification Or spec（正しい挙動に修正）
↓
task（実装 ＋ テスト）
```

### 5. 運用改善
requirement不要
```
source（運用課題）  
↓  
issue（非効率）  
↓  
decision（どうするか 主役）  
↓  
specification Or spec（ルール化，runbookに入れて，そこを参照する形でもよい）  
↓  
task（実装 ＋ テスト）
```

### 6. 技術検証（R&D）
```
source（技術・PoC）
↓  
issue（調査）
↓
decision（できること）
↓
spec（実装案）
↓
requirement（逆生成）
↓
specification（外部 場合によってはspec修正）
```

### 7. 外部仕様だけ欲しい案件（二次請け）
```
source（元請け要件）
↓
requirement（取り込み 固定）
↓
specification Or spec（外部仕様書が主役）
↓
task（実装 ＋ テスト）
```

### 8. API連携系
```
source（外部API仕様）
↓
issue（制約）
↓
decision（使い方）
↓
requirement
↓
specification Or spec（入出力中心）
↓
task（実装 ＋ テスト）
```

### 9. バッチ処理
```
source（データ・運用）
↓
issue（差分・失敗）
↓
decision（処理方式）
↓
requirement
↓
specification Or spec（ログ・状態中心 UIなし）
↓
task（実装 ＋ テスト）
```

### 10. セキュリティ対応
```
source（規約・監査）
↓
issue（リスク）
↓
decision（対策）
↓
requirement（必須事項 ここが強い）
↓
specification Or spec（アクセス制御）
↓
task（実装 ＋ テスト）
```

### 11. パフォーマンス改善

```
source（遅い）
↓
issue（ボトルネック）
↓
decision（改善方法）
↓
specification Or spec（外部仕様はほぼ触らない）
↓
design変更
↓
task（実装 ＋ テスト）
```

### 12. データ設計変更
```
source（データ不整合）
↓
issue
↓
decision
↓
specification Or spec（designでER図とか変える 完全内部）
↓
design（DB）
↓
task（実装 ＋ テスト）
```

### 13. 運用トラブル再発防止
```
source（障害）
↓
issue（原因）
↓
decision（再発防止）
↓
requirement（必要なら）
↓
specification Or spec（docsのrunbookやtroubleshootingなどにいれる）
↓
task（実装 ＋ テスト）
```

### UI改善
```
source（使いづらい）
↓
issue
↓
decision
↓
requirement
↓
specification Or spec（画面）
↓
task（実装 ＋ テスト）
```

```
decision

task（実装 ＋ テスト）
```

## 状態と要所での作成
全状態として「UNFINISHED（中途半端）」がある。（source以外）
これは、上記の経路で一括作成した際に、まだなにも書ききれていないことを表すパラメータである。

| 種類(プレフィックス)  | 状態            | タイミング  | タイミング詳細                         |
| ------------ | ------------- | ------ | ------------------------------- |
| SRC          | NEW           | 思いつき   | IDEA、DEV、MTGで出た瞬間               |
| SRC          | TRIAGED       | 分類     | Task / Decision / Issue に振り分け検討 |
| SRC          | PROMOTED      | 昇格     | Decision / Task / Issue に変換     |
| SRC          | DROPPED       | 破棄     | やらないと決定                         |
| ISSUE        | UNFINISHED    | 未整理    | まだ作成していない(課題存在も不明)              |
| ISSUE        | OPEN          | 問題発見   | バグ・不具合・課題を認識                    |
| ISSUE        | INVESTIGATING | 調査開始   | 原因調査を開始                         |
| ISSUE        | RESOLVED      | 解決     | 原因特定・対応完了                       |
| ISSUE        | CLOSED        | 完全終了   | 再発可能性も含めてクローズ                   |
| DEC          | UNFINISHED    | 未整理    | 判断途中・まだ整理できていない                 |
| DEC          | DRAFT         | 検討     | 案があるが未確定                        |
| DEC          | ACTIVE        | 決定     | 採用決定                            |
| DEC          | SUPERSEDED    | 置き換え   | 新しいDecisionに置き換えられた             |
| DEC          | DEPRECATED    | 廃止     | 不要・無効化                          |
| REQ          | UNFINISHED    | 未整理    | 要件として未整理                        |
| REQ          | DRAFT         | 曖昧・合意前 | 初期要件                            |
| REQ          | ACTIVE        | 合意     | 外部と合意済み                         |
| REQ          | DEPRECATED    | 不要     | 仕様変更等で削除                        |
| (INT_)SPEC   | UNFINISHED    | 未整理    | (内部仕様未整理)テンプレのみ・未記述             |
| (INT_)SPEC   | DRAFT         | 作成中    | (ロジック検討中)要件から作成中                |
| (INT_)SPEC   | ACTIVE        | 正式仕様   | (実装方針として確定)実装と一致・Single Source  |
| (INT_)SPEC   | NEED_UPDATE   | ズレ発生   | 実装との差異あり                        |
| (INT_)SPEC   | DEPRECATED    | 廃止     | 不要になった仕様                        |
| (INT_)DESIGN | UNFINISHED    | 未整理    | 設計未作成                           |
| (INT_)DESIGN | DRAFT         | 作成中    | 設計検討中                           |
| (INT_)DESIGN | ACTIVE        | 採用     | 実装構造として確定                       |
| (INT_)DESIGN | NEED_UPDATE   | ズレ発生   | 実装との差異あり                        |
| (INT_)DESIGN | DEPRECATED    | 廃止     | 不要になった                          |
| TASK         | UNFINISHED    | 未整理    | TODO未作成                         |
| TASK         | TODO          | 作業発生   | Decision / Designから生成           |
| TASK         | DOING         | 作業中    | 実装中                             |
| TASK         | BLOCKED       | 停止     | 依存関係・問題で進行不可                    |
| TASK         | DONE          | 完了     | 実装・確認完了                         |
| TASK         | CANCELLED     | 中止     | 不要・中断                           |
| TEST         | UNFINISHED    | 未整理    | テスト未設計                          |
| TEST         | DRAFT         | 作成中    | テスト設計中                          |
| TEST         | READY         | 実行可能   | テスト準備完了                         |
| TEST         | PASSED        | 成功     | テスト成功                           |
| TEST         | FAILED        | 失敗     | テスト失敗                           |
| TEST         | DEPRECATED    | 廃止     | 不要になったテスト                       |
